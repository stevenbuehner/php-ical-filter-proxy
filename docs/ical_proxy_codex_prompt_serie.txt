CODEX PROMPT-SERIE
iCal Proxy & Filter Service
PHP 8.3+, Composer, YAML, sabre/vobject, Symfony-Komponenten

Ziel dieser Prompt-Serie:
Mit diesen Prompts soll Codex Schritt für Schritt ein leichtgewichtiges PHP-Projekt erstellen, das externe iCal-Feeds lädt, per YAML konfiguriert, filtert, merged, cached und als neue kryptische iCal-Abo-Links ausgibt.

Wichtige Grundregeln für alle Prompts:
- Arbeite mit PHP 8.3+.
- Verwende Composer.
- Verwende sabre/vobject für iCalendar Parsing und Serialisierung.
- Verwende Symfony-Komponenten, aber kein schweres Full-Stack-Symfony-Projekt.
- Schreibe modernen, gut typisierten PHP-Code.
- Nutze strict_types=1.
- Verwende klare Klassen, Interfaces und DTOs.
- Achte auf Testbarkeit und spätere Erweiterbarkeit für eine Admin-GUI.
- Implementiere die Features schrittweise.
- Baue keine Admin-GUI, aber bereite die Struktur dafür vor.
- Die YAML-Konfiguration ist die zentrale Datenquelle.
- Die YAML-Konfiguration muss vor Nutzung validiert werden können.
- Filterregeln arbeiten sequenziell.
- Jede Filterregel hat genau eine action: keep oder remove.
- Standard für action ist remove, wenn action fehlt.
- action: remove entfernt passende Events.
- action: keep entfernt alle nicht passenden Events.
- Ein Export kann mehrere Quellen zusammenführen.
- Doppelte Events werden anhand UID entfernt. Strategie: first wins.
- Caching existiert auf zwei Ebenen: Source-Cache und Export-Cache.

PROMPT 2: YAML-KONFIGURATION UND DTO-MODELL
============================================================

Implementiere das Konfigurationsmodell für die YAML-Datei.

Die YAML-Struktur soll ungefähr so aussehen:

sources:
  gemeinde:
    label: "Gemeindekalender"
    url: "https://example.com/calendar.ics"
    cache_ttl: "15m"
    filters: []

exports:
  technikdienst:
    title: "Technikdienst"
    slug: "technikdienst"
    token: "random-secret-token"
    cache_ttl: "10m"
    include_sources:
      - source: gemeinde
        filters:
          - name: "Nur Techniktermine"
            action: keep
            match:
              summary:
                contains: "Technik"

Anforderungen:

   - SourceConfig
   - ExportConfig

2. Verwende readonly Properties, wo sinnvoll.
3. ConfigLoader lädt config/calendars.yaml und gibt ein AppConfig-Objekt zurück.
4. Unterstütze optionale Filter auf Source-Ebene und Export-Include-Ebene.
5. Standardwert für action ist remove.
6. TTL-Werte bleiben zunächst als String erhalten, werden aber später validiert.
7. Unbekannte YAML-Felder sollen später durch den Validator bemängelt werden.
8. Schreibe die Klassen so, dass sie später gut für eine Admin-GUI serialisierbar wären.

    - summary

    - location
    - url
    - categories
    - date
15. Match-Operatoren:
    - contains
    - contains_any
    - contains_all
    - not_contains
    - equals
    - not_equals
    - regex
    - empty
16. Regex-Ausdrücke müssen gültig sein.
17. TTL-Werte müssen gültig sein:
    - Sekunden: 30s
    - Minuten: 15m
    - Stunden: 1h
    - Tage: 1d
18. Cache-Verzeichnisse var/cache/feeds und var/cache/exports müssen existieren und beschreibbar sein.

20. Der CLI-Befehl soll alle Fehler gesammelt ausgeben und mit Exit-Code 1 beenden.

PROMPT 8: FILTERENGINE MIT SEQUENZIELLER KEEP/REMOVE-LOGIK
============================================================

Implementiere die FilterEngine.

Anforderungen:
1. Filterregeln werden exakt in YAML-Reihenfolge ausgeführt.
2. Jede Regel hat action keep oder remove.
3. Fehlt action, gilt remove.
4. action remove:
   - Entferne alle Events, die matchen.
5. action keep:
   - Entferne alle Events, die NICHT matchen.
6. Nach jeder Regel ist die Event-Menge kleiner oder gleich.
7. FilterEngine soll Statistiken sammeln:
   - events_before
   - events_after
   - per_rule_removed_count
   - per_rule_kept_count
8. Rückgabeobjekt:
   - filteredEvents
   - statistics
9. Regeln mit leerem match sollen keine Events verändern, aber eine Warnung in den Statistiken erzeugen.
10. Source-Filter und Export-Filter sollen mit derselben Engine laufen können.

Gib mir alle neuen und geänderten Dateien vollständig aus.
